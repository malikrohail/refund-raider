# Production Sequence

This is the cross-pack execution order for turning one chosen concept into a finished submission video.

## Phase 1 — Lock the concept

- choose one concept folder
- choose one hero merchant
- choose one damaged, wrong, or missing-item case
- freeze the judged submission first

Skills:

- `blueprint`
- `content-engine`

## Phase 2 — Lock the proof loop

- verify Firecrawl returns usable evidence
- verify ElevenLabs gives one strong explanation clip
- verify the draft/send flow is visually legible
- verify the timeline ends cleanly

Skills:

- `search-first`
- product-side Firecrawl + ElevenLabs

## Phase 3 — Capture assets

- record the problem input
- record the Firecrawl evidence state
- record the ElevenLabs explanation moment
- record the draft state
- record approve/send
- record the timeline end state

Skills:

- `video-editing`
- `mediabunny`

## Phase 4 — Build supporting audio

- generate stitched narration
- generate one or two alternate lines if needed
- generate background score variants

Skills:

- `elevenlabs-remotion`
- `text-to-speech`
- `elevenlabs-music`

## Phase 5 — Compose the master cut

- place scenes in Remotion
- add overlays, captions, timer, or waveform only if the concept needs them
- export the judged submission version first

Skills:

- Remotion
- `video-editing`

## Phase 6 — Create platform variants

- make TikTok / Reels cut
- make X-native cut
- make LinkedIn version

Skills:

- `content-engine`
- `crosspost`
- `article-writing`

## Phase 7 — Final polish

- visual QA
- cover-frame sanity check
- caption readability check
- audio mix check

Skills:

- `design-review`
- `mediabunny`

## Rule

Do not build all five videos. Build one judged submission first, then derive variants.
