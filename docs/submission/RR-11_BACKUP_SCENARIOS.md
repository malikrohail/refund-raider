# RR-11 Backup Scenarios

Use these only if the primary hero scenario is unavailable during the live demo.

## Backup 1: Wrong Item Delivered

- User ordered one product and received a different one.
- Policy usually has a clean written return or replacement path.
- Best when you need a simple mismatch story.

## Backup 2: Damaged Electronics Within Return Window

- Strong because the policy window is easy to show.
- Good for visible proof such as a photo or box damage.
- Use when you want the strongest evidence-to-verdict bridge.

## Backup 3: Missing Item, Support Not Responding

- Works when there is a clear order record and a public support email.
- Good fallback if the merchant’s policy is broad but the response is delayed.
- If needed, end on `manual submit required` while still showing the drafted request.

## Backup Selection Rule

Choose the scenario that still supports:

1. public policy evidence
2. a visible support path
3. a believable email send
4. a clear status timeline
