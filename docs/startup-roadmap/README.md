# Startup Roadmap

## Phase 1: Turn the Demo Into a Real Product Base

- keep the current refund flow working
- make voice the primary shell
- ship the generalized action-engine records and APIs
- make E2E run from a cold checkout

## Phase 2: Launch Narrow, Look Broad

Ship these playbooks first:

1. ecommerce refund
2. replacement for wrong or damaged item
3. subscription cancellation with refund recovery
4. ignored-support follow-up escalation

Broad consumer positioning is fine.
Broad fully automated coverage is not required on day one.

## Phase 3: Ops-Backed Autopilot

- use the new action plan to decide whether a case is safe for automation
- route unsupported or low-confidence cases into an ops lane
- keep the user in one coherent experience instead of exposing system gaps

## Phase 4: Merchant Learning Loop

- capture which merchants work well by email
- capture which merchants require forms, portals, or special playbooks
- turn repeated wins into reusable merchant profiles and action patterns

## Success Metrics

- time to first action
- action completion rate
- approval conversion rate
- ops fallback rate
- merchant response rate
- recovered dollar volume
- repeat case creation by returning users

## Product Rule

Do not sell “literally anything” as fully automated on day one.
Sell one strong agent experience that can handle a broad set of consumer problems through:

- direct automation when confidence is high
- ops-backed fallback when confidence is low
- clear evidence and approvals throughout
